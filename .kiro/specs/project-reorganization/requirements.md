# Requirements Document

## Introduction

Проект анализа данных Divvy bikes нуждается в полной реорганизации структуры файлов и папок. В корневой директории находится множество файлов изображений, анимаций и отчетов, которые должны быть правильно структурированы согласно стандартам data science проектов. Необходимо навести порядок, удалить неиспользуемые папки и создать четкую иерархию файлов.

## Requirements

### Requirement 1

**User Story:** Как разработчик, я хочу иметь четкую структуру папок для всех визуализаций, чтобы легко находить нужные файлы и поддерживать проект.

#### Acceptance Criteria

1. WHEN проект реорганизован THEN все PNG файлы SHALL быть перемещены в соответствующие подпапки в reports/figures/
2. WHEN проект реорганизован THEN все GIF анимации SHALL быть перемещены в reports/figures/animations/
3. WHEN проект реорганизован THEN все HTML отчеты SHALL быть перемещены в reports/
4. WHEN файлы перемещены THEN корневая директория SHALL содержать только основные конфигурационные файлы и папки

### Requirement 2

**User Story:** Как аналитик данных, я хочу иметь логическую группировку визуализаций по типам анализа, чтобы быстро находить нужные графики.

#### Acceptance Criteria

1. WHEN визуализации организованы THEN файлы SHALL быть сгруппированы по типам: heatmaps, seasonal, hourly, station_analysis
2. WHEN файлы сгруппированы THEN каждая группа SHALL иметь свою подпапку в reports/figures/
3. WHEN структура создана THEN файлы с похожими названиями SHALL быть в одной группе
4. IF файл содержит "heatmap" в названии THEN он SHALL быть в папке reports/figures/heatmaps/

### Requirement 3

**User Story:** Как участник команды, я хочу удалить неиспользуемые и дублирующиеся папки, чтобы проект был чистым и понятным.

#### Acceptance Criteria

1. WHEN проект очищен THEN неиспользуемые папки SHALL быть удалены
2. WHEN папки проверены THEN дублирующиеся структуры SHALL быть объединены или удалены
3. WHEN очистка завершена THEN backup папки SHALL быть удалены если они не содержат уникальных данных
4. IF папка пустая или содержит только временные файлы THEN она SHALL быть удалена

### Requirement 4

**User Story:** Как разработчик, я хочу иметь правильную структуру согласно cookiecutter data science template, чтобы проект соответствовал стандартам.

#### Acceptance Criteria

1. WHEN структура создана THEN она SHALL соответствовать cookiecutter data science template
2. WHEN папки созданы THEN каждая папка SHALL иметь четкое назначение
3. WHEN реорганизация завершена THEN README.md SHALL быть обновлен с актуальной структурой
4. IF существуют конфликты между старой и новой структурой THEN приоритет SHALL быть у новой структуры

### Requirement 5

**User Story:** Как пользователь проекта, я хочу иметь понятную документацию новой структуры, чтобы легко ориентироваться в проекте.

#### Acceptance Criteria

1. WHEN реорганизация завершена THEN SHALL быть создан файл STRUCTURE.md с описанием новой организации
2. WHEN документация создана THEN она SHALL содержать описание каждой папки и ее назначения
3. WHEN файлы перемещены THEN SHALL быть создан migration log с информацией о перемещениях
4. IF файлы были переименованы THEN это SHALL быть задокументировано
