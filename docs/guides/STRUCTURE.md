# Project Structure Guide

Этот документ описывает организацию проекта Divvy Bikes Analysis после реорганизации.

## Обзор структуры

Проект следует стандартам cookiecutter data science template с дополнительными улучшениями для лучшей организации файлов.

## Основные директории

### `/src/` - Исходный код
Содержит весь производственный код проекта, организованный по функциональным модулям:

- **`config/`** - Конфигурационные файлы и настройки
- **`data/`** - Модули для загрузки, очистки и обработки данных
- **`features/`** - Feature engineering и создание признаков
- **`models/`** - Модели машинного обучения и аналитики
- **`visualization/`** - Код для создания визуализаций

### `/notebooks/` - Jupyter Notebooks (Организованные)
Notebooks теперь структурированы по типам деятельности:

#### `exploratory/` - Исследовательские ноутбуки
- Первичное изучение данных
- Быстрые эксперименты
- Прототипирование
- Файлы: `start.ipynb`, `quick_start.ipynb`, `data_profiling_example.ipynb`

#### `analysis/` - Анализ данных
- Анализ по годам (2020-2025)
- Проверка гипотез
- Детальный анализ паттернов
- Файлы: `2020.ipynb`, `2021.ipynb`, `гипотизы.ipynb`, `анализ витрин.ipynb`

#### `modeling/` - Машинное обучение
- Разработка и обучение моделей
- Юнит-экономические модели
- Файлы: `модели.ipynb`, `юнит модель.ipynb`

#### `visualization/` - Визуализация
- Создание презентаций
- Дашборды и интерактивные визуализации
- Файлы: `преза.ipynb`

#### `reports/` - Отчеты
- Финальные отчеты
- Обработка данных для экспорта
- Файлы: `в csv.ipynb`, `заполнение пропусков по всем годам.ipynb`

#### `experiments/` - Эксперименты
- Тестирование новых подходов
- A/B тесты
- Временные эксперименты

#### `assets/` - Вспомогательные файлы
Организованное хранение файлов, созданных в notebooks:

- **`models/`** - Сохраненные модели (.pkl файлы)
- **`data/`** - CSV, JSON, TXT файлы с результатами
- **`images/`** - PNG, GIF изображения и графики
- **`html/`** - HTML карты и интерактивные отчеты

### `/reports/` - Финальные отчеты и визуализации
Организованное хранение всех визуализаций и отчетов:

#### `figures/` - Визуализации по категориям
- **`animations/`** - GIF анимации (38 файлов)
- **`heatmaps/`** - Тепловые карты (12 файлов)
- **`seasonal/`** - Сезонный анализ (360 файлов)
- **`hourly/`** - Почасовой анализ (48 файлов)
- **`stations/`** - Анализ станций
- **`trends/`** - Трендовый анализ
- **`combined/`** - Комбинированные визуализации (24 файла)
- **`spatial/`** - Пространственный анализ (72 файла)
- **`other/`** - Прочие визуализации

#### HTML отчеты (корень `/reports/`)
- Интерактивные презентации
- Аналитические отчеты
- Карты и дашборды

### `/models/` - Основные модели
Сохраненные модели для производственного использования.

### Другие директории

- **`/tests/`** - Тесты кода
- **`/docs/`** - Документация проекта
- **`/references/`** - Справочные материалы
- **`/scripts/`** - Скрипты автоматизации
- **`/hasura-cloud-project/`** - GraphQL проект
- **`/map_cache/`** - Кэш карт

## Принципы организации

### 1. Разделение по типам деятельности
- Исследование → `notebooks/exploratory/`
- Анализ → `notebooks/analysis/`
- Моделирование → `notebooks/modeling/`
- Визуализация → `notebooks/visualization/`
- Отчеты → `notebooks/reports/`

### 2. Логическая группировка файлов
- Файлы группируются по смыслу и назначению
- Похожие файлы находятся в одной папке
- Четкая иерархия папок

### 3. Разделение кода и артефактов
- Код → `/src/`
- Исследования → `/notebooks/`
- Результаты → `/reports/`
- Модели → `/models/` и `/notebooks/assets/models/`

## Рекомендации по использованию

### Для новых ноутбуков:
1. **Исследование данных** → `notebooks/exploratory/`
2. **Анализ конкретного года/периода** → `notebooks/analysis/`
3. **Разработка моделей** → `notebooks/modeling/`
4. **Создание визуализаций** → `notebooks/visualization/`
5. **Финальные отчеты** → `notebooks/reports/`

### Для сохранения файлов:
1. **Модели (.pkl)** → `notebooks/assets/models/`
2. **Данные (.csv, .json)** → `notebooks/assets/data/`
3. **Изображения (.png, .gif)** → `notebooks/assets/images/`
4. **HTML файлы** → `notebooks/assets/html/`

### Для финальных результатов:
1. **Визуализации** → соответствующая папка в `reports/figures/`
2. **HTML отчеты** → `reports/`
3. **Производственные модели** → `models/`
